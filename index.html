git使用日记

2018-01-19

git常用命令

为了方便自己以后使用git命令，避免网上查阅，所以用这个记录了一些常用的命令：

  1. git status 查看仓库当前状态
  2. git add xx 提交文件xx到暂存区
  3. git diff xx 查看xx文件做了什么修改
  4. git commit -m “描述” 提交修改时带上描述，将暂存区提交到当前分支
  5. git log 查看commit（提交）历史记录 可加上 –pretty=oneline 过滤其它信息，加上--graph
  6. git reset --hard HEAD^ 回退到上一版本，如果有多个版本，可以使用HEAD^^（回退上上个版本），HEAD~100（回退到100个版本）
  7. git reset --hard xxxxxxx 指定回到某个版本号
  8. git reflog 记录每一次命令操作
  9. git checkout -- xx 撤销对xx文件的全部修改操作；一种是readme.txt自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；一种是readme.txt已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。
  10. git rm xx 从版本库删除xx文件
  11. git clone git@www.xx.com:/home/xx.git 克隆到本地
  12. git branch zz 创建zz分支
  13. git checkout zz 切换到zz分支
  14. git checkout -b zz 创建并切换到zz分支，是上面两条的简写，如果要将远程的zz分支同步下来,则需要:git checkout -b zz origin/zz
  15. git branch 列出所有分支，当前分支前会有*
  16. git merge zz 将zz分支合并到当前分支
  17. git branch -d zz 删除zz分支
  18. git merge --no-ff -m “信息” zz 将zz分支合并到当前分支，并保留zz分支信息
  19. git stash 将工作现场存储起来，以后可以恢复
  20. git stash list 列举保存的工作现场
  21. git stash apply stash@{0} -> git stash drop stash@{0} 先将工作现场0恢复，然后删除工作现场0
  22. git stash pop 恢复的同时删除现场
  23. git remote 查看远程仓库信息，加上-v显示更详细信息
  24. git push origin zz 向远程分支推送zz分支
  25. git pull 将最新的提交从远程抓下来，在本地进行合并后再提交
  26. git branch –-set-upstream zz origin/zz 设置zz分支对应远程仓库的zz分支
  27. git tag v1.0 在当前分支上创建v1.0标签，加上 5134234参数代表在这个commit上打标签
  28. git tag 查看所有标签
  29. git show v1.0 查看v1.0标签信息
  30. git tag -a v1.0 -m “这是v1.0版本” 21312在当前分支的21312commit上添加标签名为v1.0，内容为”这是v1.0版本”的标签，-a指定标签明，-m指定说明文字

angular2子路由跳转

2017-10-22

配置了路由表，并定义了子路由，但是每次跳转到子路由总是跳到404路由

在angular的官方文档找到了原因：导入模块的时候需要注意顺序，我在根模块导入模块时先把主路由模块导入了，导致路由会率先匹配到通配符路由，所以跳转到404，在定义子路由时，要将子路由所在模块提前于通配符路由所在的模块！
